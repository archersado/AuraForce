# Step 1: Prototype Planning

Define prototype scope, fidelity level, and prepare for user testing.

让我加载Design Sprint的工作成果...**""

### 2. Define Prototype Scope

**确定关键用户流程：**

From Design Sprint user journeys, identify:
1. **核心用户流程**:
   - User Registration
   - User Login
   - Main Dashboard
   - [Key Feature flow]

2. **关键交互**:
   - Form submissions
   - Navigation patterns
   - Error handling
   - Success states

### 3. Select Fidelity Level

**选择原型保真度：**

```
需求评估:
- 时间约束: [紧张/充足]
- 测试类型: [早期验证/最终测试]
- 技能水平: [需要学习/熟练]
- 协作需求: [内部团队/外部测试]

推荐保真度级别:
- 概念级测试 → 低保真 (线框图+基础交互)
- 设计评审 → 中保真 (视觉设计+主要交互)
- 用户测试 → 高保真 (完整视觉+详细交互)
```

### 4. User Testing Plan

**制定用户测试计划：**

```
测试目标: [明确测试目标]

测试参与者:
- 目标用户: [用户画像]
- 数量: [5-8位]
- 招募渠道: [渠道]

测试场景:
1. [场景1]: 用户首次注册并完成核心任务
2. [场景2]: 用户使用主要功能
3. [场景3]: 遇到错误时的处理

测试环境:
- 地点: [远程/现场]
- 工具: [Figma Prototype/InVision]
- 时长: [每人30-45分钟]
```

### 5. Testing Script Preparation

**准备测试脚本：**

```markdown
# 测试引入
"感谢你参与这次测试。我们将测试一个[产品类型]的原型。
 这不是对你能力的测试，我们在测试设计本身。
 请畅所欲言，告诉我们你的想法。"

# 预测试问题
1. 你平时的[相关产品]使用经验是什么？
2. 你希望这个产品解决什么问题？

# 测试任务
任务1: "请尝试注册一个新账户"
任务2: "请[特定操作]"
任务3: "请找到并查看[特定功能]"

# 后测试问题
1. 最喜欢/最不喜欢什么？
2. 什么地方让你困惑？
3. 你会向朋友推荐吗？为什么？
```
