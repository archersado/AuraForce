# 创意验证检查清单

## Step 1: 初始化
- [ ] 检查现有的工作流文档
- [ ] 创建或确认输出文档位置
- [ ] 初始化frontmatter元数据
- [ ] 向用户展示欢迎信息

## Step 2: 创意捕获
### 产品概念
- [ ] 产品是什么？解决什么问题？
- [ ] 灵感来源是什么？
- [ ] 为什么这个产品重要？

### 核心功能
- [ ] 最核心的功能是什么？
- [ ] 最基本的操作是什么？
- [ ] 只能保留一个功能是什么？

### 目标用户（初步）
- [ ] 理想用户是什么样的？
- [ ] 主要为谁设计？
- [ ] 谁会最先使用？

### 动机与愿景
- [ ] 为什么选择做这个？
- [ ] 希望实现什么愿景？
- [ ] 成功的定义？

### 当前状态
- [ ] 产品处于什么阶段？
- [ ] 已有什么准备？
- [ ] 有什么时间规划？

## Step 3: 价值主张
### 目标用户深入
- [ ] 用户画像细节（年龄、职业、行业等）
- [ ] 用户当前如何解决问题？
- [ ] 最大的痛点是什么？
- [ ] 深夜困扰用户的问题？

### 痛点（Pains）
- [ ] 当前方案最不满意的地方
- [ ] 面临的最大障碍
- [ ] 花费了太多什么（时间/金钱/精力）
- [ ] 什么时候想"一定有更好的方法"

### 期望收益（Gains）
- [ ] 成功后用户的一天有什么不同
- [ ] 最希望节省或获得什么
- [ ] 理想情况应该有多简单
- [ ] 用户会感到自豪吗？为什么？

### 独特价值
- [ ] 核心差异是什么
- [ ] 选择这个产品的主要原因
- [ ] 能做到别人做不到的
- [ ] 这个差异可以被复制吗

### 价值假设
- [ ] 格式化价值主张陈述
- [ ] 验证陈述准确性

## Step 4: 可行性评估
### 技术可行性（1-10分）
- [ ] 所需技术有哪些？
- [ ] 技术复杂度如何？
- [ ] 技术挑战是什么？
- [ ] 是否可以解决？

### 资源可获得性（1-10分）
- [ ] 需要哪些技能？
- [ ] 有多少时间投入？
- [ ] 需要资金吗？
- [ ] 需要团队吗？

### 时间可行性（1-10分）
- [ ] MVP需要多长时间？
- [ ] 什么时候有测试版？
- [ ] 可以分阶段实现吗？
- [ ] 最小可行标准是什么？

### 风险评估（1-10分）
- [ ] 最大风险是什么？
- [ ] 什么导致失败？
- [ ] 担心什么？
- [ ] 如何解决困难？

## Step 5: 市场验证
### 市场需求
- [ ] 怎么知道有需求？
- [ ] 和潜在用户讨论过吗？
- [ ] 有什么证据？
- [ ] 如何解决当前问题？

### 竞争分析
- [ ] 直接竞争对手有哪些？
- [ ] 间接竞争对手有哪些？
- [ ] 现有方案的问题？
- [ ] 为什么选择你的？

### 市场规模
- [ ] 有多少人受益？
- [ ] 市场增长情况？
- [ ] 目标市场是整个还是细分？
- [ ] TAM/SAM/SOM估算？

### 市场趋势
- [ ] 现在环境有利吗？
- [ ] 有什么趋势影响？
- [ ] 为什么是现在的好时机？
- [ ] 太早或太晚的风险？

### 竞争优势
- [ ] 独特优势是什么？
- [ ] 可以被复制吗？
- [ ] 有什么壁垒？
- [ ] 竞争对手如何反击？

## Step 6: MVP范围
### MVP目标
- [ ] 核心假设是什么？
- [ ] 用户最强烈感受是什么？
- [ ] 什么让MVP成功？
- [ ] 如何验证假设？

### 功能优先级
- [ ] 绝对必须的功能（1-2个）
- [ ] 没有它无法存在的
- [ ] 从1个功能开始选什么
- [ ] Nice to have功能

### 用户旅程
- [ ] 完整旅程需要什么步骤
- [ ] 最简单的Happy Path
- [ ] 哪些步骤可以简化/省略
- [ ] 可以跳过注册或设置吗？

### 成功指标
- [ ] 希望多少用户试用
- [ ] 什么行为表示获得价值
- [ ] 什么用户反馈
- [ ] 如何衡量有效性

### 行动计划
- [ ] 接下来30天做什么
- [ ] 需要先做什么
- [ ] 什么会阻碍进度
- [ ] 什么时候有测试版
