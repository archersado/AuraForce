## 🎯 评审焦点

### 1. PRD 评审要点

- [ ] 产品概述和目标是否清晰
- [ ] 用户需求是否完整（8 个用户故事）
- [ ] 功能需求是否详细（4 个核心功能）
- [ ] 数据模型是否合理（WorkflowSpec, WorkflowFavorite）
- [ ] API 端点设计是否完整（9 个 API）
- [ ] 用户流程是否合理
- [ ] 产品路线图是否清晰
- [ ] 成功指标是否可测量
- [ ] 风险识别是否充分

**主要问题：**
1. 工作流集成到 Workspace 的用户流程是否合理？
2. 工作流集成到 Claude Code 的入口设计是否合适？
3. 工作流市场 2.0 的功能是否完整？
4. 统一设计系统的应用范围是否覆盖所有页面和组件？

### 2. UI/UX 设计评审要点

- [ ] 设计目标是否清晰
- [ ] 设计规范是否完整（colors, typography, spacing, borderRadius, shadow, transition）
- [ ] 设计系统是否与主应用一致
- [ ] 页面设计是否合理（3 个主要页面）
- [ ] 组件设计是否详细（7 个核心组件）
- [ ] 交互设计是否清晰（3 个主要交互）
- [ ] 响应式设计是否完整
- [ ] 可访问性设计是否满足 WCAG AA
- [ ] 动画效果是否合理

**主要问题：**
1. 工作流市场 2.0 的布局是否合理？是否需要调整？
2. Workspace 新建项目的工作流选择器是否易用？
3. Claude 工作流选择面板的交互方式是否合适？
4. 设计系统与现有组件的兼容性如何？
5. 深色模式的实现是否完整？

### 3. 技术可行性评估要点

#### Frontend 评估
- [ ] 基于 Tailwind CSS 的设计是否可行
- [ ] 新组件的实现复杂度是否可控
- [ ] 响应式设计的技术实现是否可行
- [ ] 状态管理（React Query、Zustand）是否足够
- [ ] 现有代码的复用性如何
- [ ] 开发所需时间是否合理

**Frontend 主要关注：**
1. 是否需要新增 React Context 或全局状态？
2. 工作流选择面板的状态如何管理（打开/关闭、选中/加载等）？
3. 工作流卡片的图片处理方式（从哪里获取？）
4. 工作流市场的分页和搜索的技术实现？
5. Claude 面板的集成方式（是否需要修改 Claude 组件）？

#### Backend 评估
- [ ] API 端点设计是否合理
- [ ] 数据模型是否需要修改
- [ ] 是否需要新增数据库表（WorkflowFavorite）
- [ ] API 的认证和权限控制是否完整
- [ ] 工作流市场的查询性能是否满足要求
- [ ] 开发所需时间是否合理

**Backend 主要关注：**
1. WorkflowFavorite 表是否必要？实现复杂度如何？
2. 工作流市场的搜索和筛选逻辑（前端过滤还是后端查询）？
3. 工作流收藏功能是否需要实时同步？
4. API 端点的错误处理是否完整？
5. 是否需要工作流使用统计功能？
