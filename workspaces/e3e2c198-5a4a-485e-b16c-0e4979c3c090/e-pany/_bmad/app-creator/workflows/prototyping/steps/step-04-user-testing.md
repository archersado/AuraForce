# Step 4: User Testing

Conduct usability testing sessions with target users.

## DIALOGUE SECTIONS:

### 1. Testing Preparation
"**🔍 用户测试**

让我们准备和进行用户测试，收集真实的用户反馈。"

### 2. Test Session Setup

**测试前准备：**

```markdown
测试环境检查:
□ 测试设备准备 (电脑/手机)
□ 浏览器兼容性 (Chrome/Safari/Edge)
□ 原型链接测试
□ 测试脚本准备
□ 录音/录像设备 (如有需要)
□ 安静的空间准备
```

### 3. Conduct Testing Sessions

**测试进行中：**

遵循最佳实践：
1. **开放式问题**: 让用户自由表达
2. **不引导**: 允许用户自己解决问题
3. **静默观察**: 记录用户行为和犹豫点
4. **追问原因**: "你为什么这么做？"
5. **保持中立**: 不影响用户选择

### 4. Observation Checklist

**观察记录表：**

| 参与者 | 任务 | 成功/失败 | 用时 | 困难点 | 评论 |
|--------|------|-----------|------|--------|------|
| P1 | 注册 | ✓ | 3:45 | 找不到邮箱验证 | 流程清晰，但邮件延迟 |
| P2 | 注册 | ✗ | - | 密码要求不清晰 | 需要实时验证提示 |
| P3 | 登录 | ✓ | 1:20 | 正常 | 顺利 |

### 5. Common Issues to Track

**常见问题分类：**

```
导航问题:
- 找不到某个页面/功能
- 不确定如何返回
- 面包屑导航缺失

交互问题:
- 点击无反应
- 意外操作
- 取消操作困难

内容问题:
- 按钮文字不清楚
- 错误消息令人困惑
- 缺少必要说明

设计问题:
- 字体太小/颜色对比度低
- 移动设备上布局错乱
- 动画性能问题
```

### 6. Feedback Collection

**收集结构化反馈：**

```markdown
用户反馈表:

参与者ID: P1
角色: [用户角色]

正面反馈:
✓ 界面美观，流程清晰
✓ 响应速度快
✓ 喜欢XX功能

负面反馈:
✗ 登录按钮不明显
✗ 错误消息不清楚
✗ 找不到设置页面

改进建议:
1. 建议增加XX功能
2. 希望简化YY流程
3. 优化ZZ页面的布局

整体评价:
- 易用性: [评分/10]
- 满意度: [评分/10]
- 推荐意愿: [是/否]
```
