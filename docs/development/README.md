# Development - 技术研发

本目录包含 AuraForce 项目的技术研发文档，包括 Epic、开发任务和技术文档。

## 📂 目录结构

### [epic/](./epic/) - 功能史诗
大的功能模块或项目目标的详细分解和设计。

### [tasks/](./tasks/) - 开发任务
具体的技术实现任务，可分配给开发人员。

### [technical/](./technical/) - 技术文档
技术规范、编码规范、技术最佳实践等。

---

## 🎯 研发流程

```
Epic 设计 (pm/tasks/epic/)
    ↓
技术设计 (architecture/design/)
    ↓
开发任务拆解 (pm/tasks/)
    ↓
开发执行 (development/tasks/)
    ↓
代码审查
    ↓
测试 (development/tests/)
    ↓
部署
```

## 📚 Epic 文档

每个 Epic 应包含以下内容：
- Epic 概述和目标
- 范围和边界
- Story 拆解
- 技术设计方案
- 依赖关系
- 时间规划

### Epic 模板

```markdown
# EPIC-[序号]: [Epic 标题]

## 概述
[Epic 的整体描述和目标]

## 范围
- 包含：[包含的功能]
- 不包含：[不包含的功能]

## Story 列表
| Story ID | 标题 | 优先级 | 状态 |
|----------|------|--------|------|
| STORY-001 | 标题 | P1 | 待开发 |

## 技术设计
[详细的技术设计方案]

## 依赖项
- 前置 Epic：[Epic ID]
- 后续 Epic：[Epic ID]

## 时间规划
- 开始：YYYY-MM-DD
- 结束：YYYY-MM-DD
- 预估工作量：X Story Points
```

---

## 📝 开发任务

每个 Task 应包含以下内容：
- 关联的 Story
- 任务类型（前端、后端、测试、部署）
- 实现步骤
- 验收标准
- 依赖项
- 估算时间

### Task 模板

```markdown
# TASK-[序号]: [任务标题]

## 关联信息
- **Story ID**: [Story ID]
- **类型**: 前端/后端/测试/部署
- **优先级**: P0/P1/P2
- **负责人**: [姓名]
- **预估时间**: X 小时

## 描述
[任务的详细描述]

## 实现步骤
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 依赖项
- [Task ID] - 相关任务
- [Epic ID] - 相关 Epic
```

## 🔧 技术规范

### 编码规范
- **语言规则:** TypeScript 编码规范
- **命名规范:** 驼峰式命名、文件命名
- **注释规范:** 代码注释和文档注释
- **测试规范:** 单元测试、E2E 测试

### Git 工作流
- **分支策略:** Git Flow
- **提交规范:** Conventional Commits
- **代码审查:** Pull Request + Code Review
- **CI/CD:** 自动化测试和部署

### 测试规范
- **单元测试覆盖率:** > 80%
- **E2E 测试:** 关键流程覆盖
- **性能测试:** 加载时间、响应时间
