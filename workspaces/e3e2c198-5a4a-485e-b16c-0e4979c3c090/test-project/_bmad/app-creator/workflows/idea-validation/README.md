# Idea Validation Workflow

## 工作流概述

Idea Validation工作流负责帮助用户验证和提炼产品创意，通过协作式对话将初始想法转化为清晰、可执行的产品方向。

## 工作流目标

- 捕获和记录产品创意
- 定义核心价值主张
- 评估技术和商业可行性
- 分析市场需求和竞争
- 界定MVP范围和下一步行动

## 工作流流程

```
创意捕获 → 价值主张 → 可行性评估 → 市场验证 → MVP范围
   ↓          ↓          ↓          ↓          ↓
 step-01    step-02     step-03     step-04    step-05
```

## 主要步骤

### Step 1: 创意捕获 (step-01-init & step-02-idea-capture)
- 检测现有工作流状态
- 捕获原始产品创意
- 记录灵感和动机
- 设置项目结构

### Step 2: 价值主张定义 (step-03-value-proposition)
- 定义核心价值主张
- 识别目标用户和市场
- 分析目标用户痛点
- 确定核心收益

### Step 3: 可行性评估 (step-04-feasibility)
- 评估技术可行性
- 分析资源需求
- 评估时间框架
- 识别风险和挑战

### Step 4: 市场验证 (step-05-market-validation)
- 评估市场需求
- 分析竞争格局
- 识别差异化优势
- 验证市场机会

### Step 5: MVP范围 (step-06-mvp-scoping)
- 定义MVP功能范围
- 确定优先级
- 制定行动路线图
- 提供下一步建议

## 输入要求

- 产品创意或初始想法
- 关于产品动机和愿景的描述
- 目标用户的基本概念（可选）
- 技术约束或偏好（可选）

## 输出交付物

```
{output_folder}/
└── idea-validation-{project_name}.md     # 创意验证报告
```

**报告内容包括**:
1. **创意摘要** - 产品创意的简要描述
2. **价值主张** - 核心价值和目标用户
3. **可行性分析** - 技术和商业可行性评估
4. **市场分析** - 市场需求和竞争分析
5. **MVP范围** - 最小可行产品功能范围
6. **行动建议** - 具体的下一步行动计划

15-30分钟（取决于创意复杂度）

### 工作流原则

1. **探索性对话** - 引导用户深入思考和探索创意
2. **结构化验证** - 使用标准化框架验证创意
3. **价值导向** - 始终关注用户价值和商业价值
4. **现实评估** - 提供诚实、客观的可行性分析
5. **行动驱动** - 以可执行的结果为目标

## 验证框架

该工作流使用以下验证框架:

### 产品-市场契合度 (PMF)
- 目标用户清晰度
- 痛点相关性
- 价值主张独特性
- 市场规模足够性

### 可行性评估
- 技术复杂度
- 资源可用性
- 时间可行性
- 成本合理性

### MVP原则
- 最小功能集合
- 快速验证迭代
- 用户反馈驱动
- 渐进式增强
