# 快速入门指南

欢迎使用 AI Dev Team 模块！本指南将帮助您快速上手。

---

## 模块简介

AI Dev Team 是一个协作式 AI 产研团队模块，通过多 AI Agent 协作自动完成从需求收集到产品交付的完整敏捷开发流程。

模块包含 6 个专业角色代理：
- **PM (项目经理)** — 需求记录、Backlog 管理、任务分配、里程碑汇报
- **Product Designer (产品设计)** — PRD 编写、需求分析、用户体验设计
- **Interaction Designer (交互设计)** — 交互流程设计、原型输出
- **Frontend Dev (前端开发)** — 前端架构、UI 实现、性能优化
- **Backend Dev (后端开发)** — 后端架构、API 设计、数据库设计
- **QA (测试)** — 测试用例设计、功能验证、Bug 管理

---

## 安装

```bash
bmad install ai-dev-team
```

安装时，您需要配置以下选项：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `project_name` | 项目名称 | {directory_name} |
| `artifacts_folder` | 项目产物存储位置 | {output_folder}/project-artifacts |
| `dev_docs_folder` | 开发文档存储位置 | dev-docs |

## 快速开始

### 第 1 步：向 PM 提出需求

安装完成后，向项目经理（PM）提出您的产品需求：

```
[PC] Project Create
```

PM 将：
1. 收集您的需求
2. 在 Linear 上创建项目和 Epic
3. 创建基础需求文档

### 第 2 步：产品设计评审

PM 协调产品设计进行需求分析和 PRD 编写：

```
[PD] Product Design
```

完成后将召开 **产品评审会议**（[PR] Product Review）。

### 第 3 步：交互设计评审

产品设计批准后，交互设计创建交互流程和原型：

```
[ID] Interaction Design
```

完成后将召开 **交互评审会议**（[IR] Interaction Review）。

### 第 4 步：任务拆解

交互评审通过后，PM 和开发团队进行任务拆解：

```
[TB] Task Breakdown
```

每个 Story 将包含：
- 需求描述
- 验收标准
- 复杂度评估
- 时间估算

### 第 5 步：研发与交付

前端和后端开发同时进行：

```
[FD] Frontend Dev
[BD] Backend Dev
```

### 第 6 步：测试验证

QA 进行功能测试并生成测试报告：

```
[TS] Test System
[TR] Test Report
```

## 关键里程碑

在开发过程中，以下节点需要用户确认：

1. **产品设计评审** — 确认产品功能范围
2. **交互评审** — 确认交互设计
3. **产品验收** — 最终交付确认

PM 会主动在这些节点汇报进度并向您请示。

---

## 项目进度查询

随时可以请求 PM 汇报当前项目进度：

```
[PR] Project Report
```

PM 将提供：
- 当前里程碑位置
- 完成进度百分比
- 预计完成时间
- 需要关注的问题

---

## Bug 管理

当 QA 发现 Bug 时：

1. QA 开辟测试环境并记录 Bug
2. PM 将 Bug 分配给对应开发角色
3. 开发人员修复 Bug
4. QA 重新验证
5. 验证通过后 Bug 关闭

整个过程由 PM 全程追踪。

---

## 需求变更处理

如果需要修改已确认的需求：

```
[RC] Requirement Change
```

PM 将：
- 评估变更影响
- 更新相关文档
- 通知团队调整计划
